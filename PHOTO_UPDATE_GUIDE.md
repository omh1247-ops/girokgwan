# 📸 사진 자동 업데이트 가이드

이 파일은 AI가 사진을 추가할 때 따라야 할 표준 절차입니다.

---

## 🎯 트리거 문구

사용자가 다음 중 하나라도 입력하면 이 가이드 절차를 자동으로 따릅니다:
- "사진업데이트"
- "사진 업데이트"

위 문구가 입력되면 AI는 별도 확인 없이 즉시 절차를 시작합니다.

---

## 📋 표준 절차

### 1️⃣ 폴더 자동 탐색

**검색 범위**
- 검색 범위: **프로젝트 루트(girokgwan 폴더) 안만 검색**
- 프로젝트 루트에서 이미지 파일 `.jpg`, `.jpeg`, `.png`, `.webp`이 들어있는 모든 폴더를 자동 검색

**제외 폴더 (절대 건드리지 않음)**
- `logo.png/` 또는 로고/브랜드 이미지가 있는 폴더
- `node_modules`, `.git` 같은 시스템 폴더
- `js/`, `scripts/`, `data/` 같은 코드 폴더
- `.claude/` (설정 폴더)
- 갤러리에 표시되지 않는 모든 폴더

**갤러리 폴더 (처리 대상)**
- `portraiture/` - 초상화 (마지막 번호: 12)
- `Product/` - 상품 사진 (마지막 번호: 09)
- `Food/` - 음식 사진 (마지막 번호: 04)
- `Moment/` - 순간 포착 (마지막 번호: 13)
- `Personal Works/` - 개인 작업 (마지막 번호: 02)

**동작**
- 각 갤러리 폴더에서 기존 파일명 패턴과 다른 새 사진 자동 식별
- 예: `IMG_1234.jpg`, `KakaoTalk_xxx.jpg`, `0417 조아라0250.jpg` 등 카메라/SNS 원본 파일명
- 새 사진이 있는 폴더만 처리 대상으로 선정

### 2️⃣ 파일명 패턴 자동 분석

각 폴더의 기존 파일명에서 패턴 자동 추출:

| 폴더 | 파일명 패턴 | 다음 번호 |
|------|-----------|--------|
| portraiture/ | `portrait{NN}.jpg` | 13 |
| Product/ | `product{NN}.jpg` | 10 |
| Food/ | `food{NN}.jpg` | 05 |
| Moment/ | `moment{NN}.jpg` | 14 |
| Personal Works/ | `personalwork{NN}.jpg` | 03 |

**동작**
- 마지막 파일의 번호 자동 인식
- 새 파일은 마지막 번호 + 1부터 두 자리 숫자로 부여
- 예: 현재 `portrait12.jpg`가 마지막 → 새 파일은 `portrait13.jpg`로 시작

### 3️⃣ 사진 압축 (🔴 가장 중요)

**절대 준수 사항**
- ⚠️ **가로세로 비율 절대 유지** (변경 금지)
- 기존 폴더 내 다른 사진들의 평균 파일 크기와 비슷한 수준으로 압축

**압축 사양**
- 최대 가로/세로: 1920px (긴 쪽 기준)
- JPEG 품질: 80~85%
- 결과 파일 크기: **200KB ~ 800KB** 범위
- 권장 도구: macOS의 `sips` 명령어

**사용 명령어 예시**
```bash
sips -Z 1920 "원본파일.jpg" --out "압축된파일.jpg" -s format jpeg -s formatOptions 85
```

### 4️⃣ 파일명 변경

**동작**
- 위의 분석한 패턴에 맞게 자동 변경
- 두 자리 숫자로 통일 (01, 02, ..., 09, 10, 11, 12, 13, ...)
- 압축된 새 파일이 최종 파일명으로 저장됨
- 원본 파일(`IMG_1234.jpg` 등)은 삭제

### 5️⃣ photos.json 자동 업데이트

**파일 위치**: `data/photos.json`

**동작**
- 해당 폴더의 카테고리 배열에 새 파일명 추가
- 예: portraiture에 새 사진 3장 추가 시
  ```json
  "portraiture": ["portrait04","portrait05",...,"portrait12","portrait13","portrait14","portrait15"]
  ```

### 6️⃣ 작업 완료 보고

체크리스트 형태로 보고:

```
✅ 처리된 폴더
- portraiture: 새 사진 3장

✅ 추가된 파일명
- portrait13.jpg
- portrait14.jpg
- portrait15.jpg

✅ 압축 결과 (전/후 크기)
- 원본 0417 조아라0250.jpg: 4.2MB → portrait13.jpg: 580KB
- ...

✅ photos.json 업데이트
- portraiture: 12개 → 15개
- 라인 2: portraiture 배열에 새 항목 3개 추가

✅ 검증 완료
- 모든 새 사진이 올바른 이름으로 생성됨 ✓
- 가로세로 비율 유지됨 ✓
- 압축 후 파일 크기 적절함 ✓
- photos.json 문법 정상 ✓
```

## 📐 사진 사이 간격 유지

- PC와 모바일 모두 columns 방식으로 일관된 간격 유지
- 간격 조정 위치: `.photo-grid { column-gap: 2px }` (현재 2px)
- CSS 변경은 절대 금지 ⛔

---

## 🚫 절대 금지 사항

1. **사진 처리 외**
   - 사진의 가로세로 비율 변경 금지
   - 기존 사진 파일 수정/덮어쓰기 금지
   - 사진 외 다른 코드, 함수, HTML 구조, CSS 변경 금지

2. **폴더/파일 처리**
   - 로고, 아이콘, 시스템 이미지가 있는 폴더는 절대 건드리지 말 것
   - 갤러리 표시용 사진 폴더(portraiture, Product, Food, Moment, Personal Works)만 처리
   - 새로운 폴더 생성 금지

3. **판단 기준**
   - 폴더 구조가 모호할 때 추측해서 진행 금지 (반드시 사용자에게 확인 요청)
   - 동시에 여러 폴더에 새 사진이 있을 때 사용자 확인 없이 한쪽만 처리 금지

4. **코드 변경 금지**
   - `index.html` 변경 금지
   - `js/render.js` 변경 금지
   - CSS 변경 금지
   - 이미지 태그 형식 변경 금지

## ⚠️ 예외 상황 처리

다음 경우에는 **작업을 중단하고 사용자에게 확인 요청**:

1. **새 사진이 없을 때**
   → "새 사진을 찾을 수 없습니다. 이미지 파일을 폴더에 추가해주세요." 알림

2. **같은 폴더에 패턴이 다른 파일이 섞여있을 때**
   → "portraiture에 서로 다른 패턴의 파일이 발견되었습니다. 어떤 패턴을 따를지 확인해주세요."

3. **압축이 실패했을 때**
   → "압축 중 오류가 발생했습니다. [오류 메시지]" 보고 후 중단

4. **기존 photos.json에 카테고리가 없을 때**
   → "photos.json에 해당 카테고리가 없습니다. 확인 후 재진행해주세요."

## 🔍 작업 후 자가 검증

체크리스트로 반드시 검증:

- [ ] 모든 새 사진이 올바른 이름으로 생성되었는지
- [ ] 가로세로 비율이 원본과 동일한지
- [ ] 압축 후 파일 크기가 적절한지 (다른 사진들과 비슷한 수준)
- [ ] photos.json이 올바르게 업데이트되었는지
- [ ] photos.json 문법이 정상인지 (JSON 유효성 검증)
- [ ] 다른 코드가 변경되지 않았는지
